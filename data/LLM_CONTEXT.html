LLM Context: Markdown to HTML Compiler Project

Quick Summary

This is a Markdown-to-HTML compiler implemented in Python. It parses Markdown with extensions (footnotes, math, definition lists, etc.) using a top-down recursive descent parser, converts it to an AST via Node objects, serializes to JSON, and renders to formatted HTML.

Entry point: python3 compiler.py <markdown_file>

Parsing Pipeline


Input Markdown String
          ↓
   Parser.parse(text)
     (2-phase process)
          ↓
Phase 1: Block Parsing (Parser.incorporate_line)
- Line-by-line processing with regex pattern matching
- Detects: headings, code blocks, lists, blockquotes, math, footnotes, definitions
- Builds hierarchical block structure
- Stores inline content as string_content for Phase 2
          ↓
Phase 2: Inline Parsing (InlineParser.parse_inlines)
- Processes stored inline content with regex tokenization
- Creates nodes for: emphasis, strong, links, images, code, math, HTML, entities
- Populates AST with semantic meaning
          ↓
   Node Tree (AST)
          ↓
  node.to_json()
          ↓
  JSON IR (Intermediate Representation)
          ↓
 HtmlRenderer.render_json_ir()
          ↓
   Final HTML Output

Key Classes & Responsibilities

Parser (parser.py)

  1. `__init__`: Initializes document node and inline parser
  2. `parse(text)`: Main entry point - orchestrates two-phase parsing
  3. `incorporate_line(line)`: Phase 1 - identifies block types and builds structure
    1. Uses block_token_pattern regex with named groups
    2. Creates and manages node hierarchy
    3. Handles block state (open/closed)
  4. `process_inlines(node)`: Triggers Phase 2 inline parsing
  5. `_close_tip_if_paragraph()`: Utility to close paragraph nodes

State Variables:

  1. doc: Root Node (document type)
  2. tip: Current open node being populated
  3. current_line: Line being processed
  4. indent: Indentation level
  5. blank: Whether line is empty
  6. line_number: For debugging/tracking

InlineParser (parser.py)

  1. `parse(block_node)`: Entry for inline processing
  2. `parse_inlines(block)`: Tokenizes block.stringcontent using `inlinetokenpattern
  3. `peek()`: Peek at current character
  4. `match(pattern)`: Match regex at current position
  5. Creates child nodes for all inline elements (text, strong, emph, links, etc.)

Supported Inline Nodes:

  1. text (plain text)
  2. strong, emph (emphasis variants)
  3. link, image (with destination property)
  4. code (inline code)
  5. mark, u, del, sub, sup (formatting)
  6. math_inline (LaTeX equations)
  7. footnote_ref (references to footnotes)
  8. emoji (emoji shortcodes)
  9. task_checkbox (with checked property)
  10. html_inline (raw HTML)
  11. hardbreak, softbreak (line breaks)

Node (node.py)

  1. Tree structure: parent, firstchild, lastchild, prev, next (doubly-linked)
  2. Properties:
    1. type: node category (heading, paragraph, etc.)
    2. string_content: raw text for block nodes (processed in Phase 2)
    3. literal: text content for leaf nodes
    4. level: heading level (1-6)
    5. destination: link/image URL
    6. title: link/image alt text
    7. custom_id: heading ID from {#id}
    8. is_open: whether block accepts new children
  3. Methods:
    1. append_child(child): Add child node
    2. unlink(): Remove from tree
    3. insert_after(sibling): Structural manipulation
    4. to_json(): Serializes to JSON string or dict

HtmlRenderer (html_renderer.py)

  1. Processes JSON IR (from node.tojson())
  2. `render_json_ir(ir_dict)`: Main entry
  3. `_render_node(node, tight=False)`: Recursive rendering
  4. `escape(text)`: HTML entity escaping (prevents XSS)
  5. Handles all node types with proper HTML tags:
    1. Block: <h1-6>, <p>, <blockquote>, <ul>, <ol>, <div>, etc.
    2. Inline: <strong>, <em>, <code>, <a>, <img>, etc.
    3. Special: Math via KaTeX, footnotes, custom IDs

Regex Patterns (regex_rules.py)

Block Token Pattern

Combined regex with named groups detecting:

  1. thematic_break: *** or --- or ___
  2. bullet_list: *, +, - items
  3. ordered_list: 1. numbered items
  4. atx_heading: # to ###### headings
  5. code_fence: or (3+ characters)
  6. math_fence: $$
  7. mermaid_fence: ::: mermaid
  8. mermaid_close: :::
  9. footnote_def: [^label]: content
  10. def_list_marker: : definition

Inline Token Pattern

Matches (in order of precedence):

  1. Images: ![alt](url)
  2. Links: [label](url)
  3. Strong: **text** or __text__
  4. Emphasis: *text* or _text_
  5. Highlight: ==text==
  6. Underline: ++text++
  7. Strikethrough: ~~text~~
  8. Subscript: ~text~
  9. Superscript: ^text^
  10. Code: code`
  11. Math: $equation$
  12. Footnote refs: [^label]
  13. Emojis: :name:
  14. Task checkboxes: [x] or [ ]
  15. HTML tags: <tag>
  16. Email autolinks: <email@example.com>
  17. URL autolinks: <https://url>
  18. HTML entities: &amp;
  19. Hard breaks: two spaces + newline
  20. Escaped chars: \*
  21. Plain text and single special chars

Special Patterns

  1. reHeadingID: {#custom-id} in headings
  2. reLineEnding: \r\n, \n, or \r normalization
  3. reHtmlBlockOpen: 7 rules for HTML block detection

Compiler Entry Point (compiler.py)


def main():
    # 1. Read markdown file from argv[1]
    # 2. Parser().parse(text) → AST
    # 3. ast.to_json() → JSON IR
    # 4. HtmlRenderer().render_json_ir(ir) → HTML body
    # 5. Load template.html
    # 6. Replace __TITLE__ and __BODY_HTML__
    # 7. Write output.<md>.html and output.<md>.json

Output Files:

  1. <name>.html: Final rendered HTML (from template)
  2. <name>.json: JSON IR for inspection/debugging

Template System (template.html)

  1. Standard HTML5 structure
  2. Placeholders: __TITLE__ (from filename) and __BODY_HTML__ (from renderer)
  3. Includes KaTeX CDN for math rendering
  4. CSS styling for body, tables, code blocks, etc.
  5. Responsive design (max-width: 800px)

Block Parsing Logic (Key Algorithm)

For each line:

  1. Strip leading whitespace → measure indent
  2. Match against `block_token_pattern` → determine block type
  3. Handle block-specific rules:
    1. Blockquote: > prefix → create/append to bq node
    2. Heading: # prefix → create heading, close paragraph
    3. Code fence: → open code block in state, wait for closing fence
    4. List item: * or 1. → create list item
    5. Footnote: [^label]: → create footnotedef node
    6. Definition: : → convert previous paragraph to def term
  4. Manage node state:
    1. Update self.tip (current open node)
    2. Set is_open = True for blocks accepting children
    3. Close blocks when new block starts at same or lower indent

Inline Parsing Logic

For each block with string_content:

  1. Tokenize using inline_token_pattern.finditer()
  2. For each token, check type by pattern:
    1. Images/links: extract label and URL, create node
    2. Formatting: check delimiters, extract content
    3. Code: count backticks, strip matching count
    4. Autolinks: detect email vs URL, add mailto: prefix
    5. Special nodes: math, emoji, footnote refs
    6. Fallback: plain text node
  3. Append nodes to parent block node

Important Design Decisions

  1. Two-phase parsing: Separates structural parsing from content parsing
  2. String deferral: Block parsing stores inline content as stringcontent, processed later
  3. Regex-based lexing: No traditional tokenizer; patterns handle tokenization
  4. Single regex per phase: Combined patterns for efficiency
  5. Named groups: Used to identify token type without secondary matching
  6. Node tree first: JSON and HTML generated from validated AST
  7. JSON intermediate: Enables non-HTML backends and inspection

Common Modifications

Add a new block type:

  1. Add pattern to block_token_pattern in regex_rules.py
  2. Add condition in Parser.incorporate_line()
  3. Create node with appropriate properties
  4. Add rendering logic to HtmlRenderer._render_node()

Add a new inline element:

  1. Add pattern to inline_token_pattern in regex_rules.py
  2. Add condition in InlineParser.parse_inlines()
  3. Extract content and create node
  4. Add rendering case in HtmlRenderer._render_node()

Change HTML output:

  1. Modify HtmlRenderer._render_node() for specific node type
  2. Adjust template.html styling

Debugging Tips

  1. Check output.json to see AST structure
  2. Verify block_token_pattern.match() output for parsing issues
  3. Use node.to_json(as_dict=True) to inspect node properties
  4. Add print statements in incorporate_line() or parse_inlines() to trace execution

Performance Notes

  1. Linear time complexity: O(n) where n = input length
  2. Single-pass block parsing
  3. Regex overhead for pattern matching (acceptable for markdown-sized inputs)
  4. No heavy dependencies (pure Python stdlib)
