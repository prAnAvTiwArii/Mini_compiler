Markdown to HTML Compiler

A Python-based Markdown compiler that parses Markdown syntax into an Abstract Syntax Tree (AST), converts it to JSON intermediate representation (IR), and renders it to HTML with extensive formatting support.

Project Overview

This project implements a top-down recursive descent parser that processes Markdown files with extensions and converts them into styled HTML output. It uses regex-based lexical analysis combined with an iterative state machine to build the parse tree.

Architecture


Input (Markdown) 
    ↓
[Parser] - Block & Inline Parsing → AST (Node Tree)
    ↓
[Node] - Tree representation with JSON serialization
    ↓
[HtmlRenderer] - JSON IR → HTML tags
    ↓
Output (HTML file + JSON IR)

Features

Block-Level Elements

  1. Headings (ATX style: # H1 to ###### H6) with custom IDs ({#custom-id})
  2. Paragraphs
  3. Blockquotes (> text)
  4. Code Blocks (fenced with or with language info)
  5. Thematic Breaks (***, ---, ___)
  6. Lists (bullet and ordered)
  7. HTML Blocks (raw HTML passthrough)
  8. Footnote Definitions ([^label]: content)
  9. Definition Lists (: definition)
  10. Math Blocks ($$..$$)
  11. Mermaid Diagrams (::: mermaid)

Inline Elements

  1. Text Formatting
    1. Bold: **text** or __text__
    2. Italic: *text* or _text_
    3. Strikethrough: ~~text~~
    4. Underline: ++text++
    5. Mark/Highlight: ==text==
    6. Subscript: ~text~
    7. Superscript: ^text^
  2. Links & Images
    1. Links: [label](url)
    2. Images: ![alt](url)
    3. Email autolinks: <user@example.com>
    4. URL autolinks: <https://example.com>
  3. Code & Math
    1. Inline code: code`
    2. Inline math: $equation$
  4. Other
    1. Footnote references: [^label]
    2. Emoji shortcuts: :emoji_name:
    3. Task checkboxes: [x] or [ ]
    4. HTML entities: &amp;
    5. Hard breaks: two spaces + newline
    6. Soft breaks: newline
    7. Escaped characters: \*

File Structure

Core Modules

  1. `parser.py` - Main parsing logic
    1. Parser class: Block-level parsing with two-phase approach (Phase 1: blocks, Phase 2: inlines)
    2. InlineParser class: Processes inline tokens using regex patterns
  2. `node.py` - AST node representation
    1. Node class: Tree structure with parent-child relationships, JSON serialization
  3. `html_renderer.py` - HTML output generation
    1. HtmlRenderer class: Converts JSON IR to styled HTML with proper escaping
  4. `regex_rules.py` - Pattern definitions
    1. Block-level token patterns
    2. Inline token patterns
    3. HTML block detection rules
    4. Special regex patterns (heading IDs, line endings)
  5. `compiler.py` - Entry point
    1. Orchestrates parsing, JSON generation, and HTML rendering
    2. Arguments: input markdown file
    3. Outputs: .html and .json files
  6. `template.html` - HTML template
    1. Base template with styling
    2. Placeholders: __TITLE__ and __BODY_HTML__
    3. Includes KaTeX CDN for math rendering

Usage


python3 compiler.py input.md

Output:

  1. input.html - Rendered HTML file
  2. input.json - JSON intermediate representation (for debugging/inspection)

Parser Type Details

This is a top-down recursive descent parser with the following characteristics:

  1. Single-pass, line-by-line processing: Processes input sequentially
  2. Two-phase parsing:
    1. Block structure parsing to create document hierarchy
    2. Inline element parsing within blocks
  3. Regex-based pattern matching: Uses combined regex with named groups
  4. State machine approach: Maintains parsing state across line processing
  5. AST generation: Builds Abstract Syntax Tree before rendering

Dependencies

  1. Python 3.6+
  2. No external packages required (uses only standard library: re, json, sys, os, html, urllib.parse)

JSON Intermediate Representation

The JSON output contains the complete AST structure:


{
  "type": "document",
  "children": [
    {
      "type": "heading",
      "level": 1,
      "children": [
        {
          "type": "text",
          "literal": "Title"
        }
      ]
    },
    ...
  ]
}

This JSON can be:

  1. Inspected for debugging
  2. Used by external tools
  3. Transformed into different output formats

Styling

The HTML output includes:

  1. Responsive design (max-width: 800px)
  2. Professional typography with system fonts
  3. Table styling with borders
  4. Code block syntax (ready for highlighting integration)
  5. Math rendering support via KaTeX

Extensions Supported

  1. Custom heading IDs: # Heading {#custom-id}
  2. Task lists: [x] Completed task
  3. Footnotes: [^1] with definitions [^1]: Footnote content

  5. Math blocks: $$formula$$
  6. Mermaid diagrams: ::: mermaid ... :::
